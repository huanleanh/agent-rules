# Claude Code Rules (CLAUDE.md)

## Workflow
- Use `ls`, `grep`, and `cat` (or `view_file`) to understand the codebase.
- When creating new rules, ensure they follow the established template.
- Use the `implementation_plan` pattern for complex additions.

- Use clear, professional language in all documentation.
- Follow the language-specific rules in the `languages/` directory.

## Commands
- Test scripts should be placed in a `tests/` directory if applicable.
- Use `npm test`, `pytest`, or equivalent for verification.
