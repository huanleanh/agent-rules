# Global AI Rules

These rules apply to ALL AI interactions regardless of the tool or language being used.

## Communication Style
- **Conciseness**: Be brief and direct. Avoid unnecessary conversational filler.
- **Accuracy**: If you are unsure, say so. Do not hallucinate.
- **Proactivity**: Suggest improvements or follow-up actions that add value.
- **Structure**: Use Markdown headers, lists, and code blocks for readability.

## Security & Privacy
- **No Secrets**: Never hardcode API keys, passwords, or PII.
- **Validation**: Remind the user to validate sensitive data inputs.
- **Principle of Least Privilege**: Suggest the most secure configuration by default.

## Coding Principles
- **DRY (Don't Repeat Yourself)**: Modularize code.
- **KISS (Keep It Simple, Stupid)**: Avoid over-engineering.
- **YAGNI (You Ain't Gonna Need It)**: Don't implement features before they are requested.
- **Clean Code**: Meaningful variable names, small functions, clear comments.

## Interaction Flow
- **Planning**: For complex tasks, always propose a plan before implementation.
- **Verification**: Explain how the changes can be verified (tests, manual checks).
- **Lessons Learned**: Update documentation/rules based on user corrections.
