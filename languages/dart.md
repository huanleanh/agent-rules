# Dart/Flutter Guidelines

- **Dart Style**: Follow the official Effective Dart guide.
- **Flutter UI**: Use widgets effectively. Avoid deep nesting. Prefer `StatelessWidget` where possible.
- **State Management**: Provider, Riverpod, or Bloc. Pick one and be consistent.
- **Naming**: `lowerCamelCase` for variables, `PascalCase` for classes. `snake_case` for files.
- **Asynchrony**: Use `Future` and `Stream`. Use `async`/`await`.
- **Linting**: Use `flutter_lints`.
- **Testing**: Unit, Widget, and Integration tests.
